# ReadMD 设计准则文档

版本：v1.1
适用范围：ReadMD Android App
更新日期：2026-05-25

## 1. 设计目标

ReadMD 的界面目标不是做成复杂的专业编辑器，而是做成一款“手机上的 Markdown 备忘录”。当前设计基调是阅读优先、编辑够用、操作明确。

核心体验优先级：

1. 能快速导入或新建文件。
2. 能清楚、舒适地阅读 Markdown 内容。
3. 需要修改时能快速进入编辑并保存。
4. 大字模式下正文、按钮和设置项都可读、可点、不重叠。
5. 日间和夜间模式都必须服务于“便于阅读”，不能出现黑底黑字、浅底浅字等低对比问题。

## 2. 原型图理解

用户给出的原型图表达的是一套轻量应用结构：ReadMD 不是展示大量工具的专业工作台，而是帮助用户快速找到一个文档、舒适阅读，并在需要时完成修改。

### 首页

- 顶部左侧是应用名 `ReadMD`。
- 顶部右侧是 `设置`。
- 主操作收敛为 `导入` 和 `新建`。
- 中间提供搜索框，用于过滤最近打开或本地笔记列表。
- 下方是最近文档/笔记列表，条目应清楚展示文件名和摘要。

首页的角色是“入口”和“文档索引”，不应堆放过多次级功能。

### 阅读页

- 打开文件或新建后，默认优先进入阅读页。
- 阅读页主体应尽量让给 Markdown 预览。
- 主要操作只保留和当前阅读任务强相关的入口，例如 `进入编辑模式`、设置入口、返回首页。
- 阅读页支持双指缩放字号，但手势只是增强操作；用户仍然可以通过设置面板滑动条调整字号。

阅读页的首要目标是让用户“看得清、看得久、不被干扰”。

### 编辑页

- 编辑页用于修改原始 Markdown 文本。
- 顶部工具栏保留 `保存`、`另存`、`导出` 和返回阅读的入口。
- 编辑框应占据页面主要高度，尤其要考虑输入法弹出后的可见空间。
- 保存或另存成功后，应自动回到阅读页，方便立即检查渲染效果。

编辑页的首要目标是“能稳定输入、能可靠保存”，不追求复杂排版编辑器能力。

### 设置面板

- 设置应作为轻量弹层或底部面板出现。
- 当前核心设置包括字号、行距、大字模式、日间/夜间模式。
- 大字模式和夜间模式互相独立，可以组合使用。
- 设置项要少而清楚，不做复杂多级配置。

## 3. 视觉语言

### 颜色

- 日间模式以白色或近白色背景、深色正文为主。
- 夜间模式以低刺激深色背景、浅色正文为主。
- 主色可使用蓝色系，但不应让界面变成单一蓝色。
- 状态色只用于必要反馈，避免杂色干扰阅读。

### 字体

- 标题层级要明显，正文保持舒适行距。
- 大字模式下正文、标题、按钮、列表项和设置项都要同步变大。
- 按钮文字变大后，按钮高度和宽度也要自适应，不能让文字重叠。

### 留白

- 首页保持轻量，不要太挤。
- 阅读页留白应服务于阅读，不要把可阅读区域压得过小。
- 编辑页留白不能牺牲输入空间，输入法场景下尤其要保证编辑框可见。

## 4. 页面结构准则

### 首页必须包含

- 应用名
- 设置入口
- 导入按钮
- 新建按钮
- 搜索框
- 最近文档/笔记列表

### 阅读页必须包含

- 文件名或当前文档标识
- 大面积 Markdown 预览区
- 进入编辑模式入口
- 返回首页能力
- 设置入口

### 编辑页必须包含

- 原始 Markdown 编辑框
- 保存
- 另存
- 导出 Markdown
- 返回阅读入口

### 设置面板必须包含

- 字号滑动条
- 行距滑动条
- 大字模式开关
- 日间/夜间模式开关

## 5. 组件准则

### 按钮

- 主按钮用于关键动作，例如导入、新建、进入编辑模式、保存。
- 次按钮用于另存、导出、返回阅读等辅助动作。
- 按钮文字要短，必要时允许换行，但不得重叠。
- 大字模式下按钮最小触控区域应随字号增大。

### 输入框

- 搜索框要足够宽，适合单手快速输入。
- 编辑框要足够大，输入法弹出后仍能看到正在输入的文字和上下文。
- 边框和背景要轻，不要压过文本本身。

### Markdown 预览区

- 阅读页预览区应是页面最大区域。
- 日间、夜间、大字日间、大字夜间都要保证正文对比度。
- 长文档滚动应稳定，手势缩放不能破坏基本滚动体验。
- 阅读页双指缩放只影响当前阅读预览，不应同步放大首页、最近文件列表或其他导航界面。
- 阅读字号上限建议控制在约 155%，避免标题、列表和大字模式组合时发生重叠。

### 设置控件

- 字号和行距使用滑动条。
- 大字模式、夜间模式使用明确按钮或开关。
- 手势缩放是阅读页增强能力，不应替代可见设置控件，也不应改变全局界面字号。

## 6. 大字模式准则

大字模式不是简单把正文放大，而是整套界面一起变舒服。

必须一起变大的内容：

- 正文
- 标题
- 按钮文字
- 列表项文字
- 设置项文字

必须一起适配的内容：

- 按钮触控区域
- 搜索框高度
- 列表项高度
- 编辑框行高
- 设置面板间距

不能出现的情况：

- 正文变大了，但按钮文字没变。
- 按钮文字变大了，但按钮尺寸没变，导致文字叠在一起。
- 大字模式只能在日间模式好用，夜间模式不可读。
- 字体和行距滑动条调整后界面没有实际变化。
- 手势放大正文后，首页或设置页被一起撑乱。

## 7. 交互准则

- 首页第一优先级是 `导入` 和 `新建`。
- 打开文档后第一优先级是 `阅读`。
- 修改内容时进入编辑页，保存或另存后回到阅读页。
- 成功打开文件不需要底部弹窗打扰阅读；失败、保存、导出等需要反馈的动作可以保留提示。
- 支持系统返回手势：设置面板先关闭，编辑页回阅读页，阅读页回首页。
- 手势功能必须是可选增强，不应让用户必须记住隐藏操作才能完成基本任务。
- 阅读页手势缩放应保持局部、可逆、低干扰；返回首页后首页布局应保持稳定。

## 8. ReadMD 应保持的风格

关键词：

- 轻
- 清楚
- 可读
- 低学习成本
- 适老化
- 本地优先
- 阅读优先

## 9. 不建议的方向

- 复杂工具栏
- 多层嵌套卡片
- 过度装饰
- 信息密度过高的页面
- 像代码编辑器一样的复杂布局
- 为了“专业感”堆太多按钮
- 让隐藏手势承担核心功能

## 10. 后续实现时的判断标准

以后每次做 UI 时，都可以问：

1. 用户能不能在 3 秒内找到导入或新建？
2. 用户打开文档后是否第一眼看到正文？
3. 用户能不能在一眼内找到进入编辑模式和保存？
4. 大字模式下按钮和正文是否都清楚、不重叠？
5. 夜间模式是否所有区域都是深底浅字，且足够柔和？
6. 输入法弹出后，编辑框是否仍然可见、可用？
7. 这个控件是否真的服务于阅读、编辑或保存？

如果答案有一个是否定的，就应该重新收敛设计。
