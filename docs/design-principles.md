# ReadMD 设计准则文档

版本：v1.0
适用范围：ReadMD Android App

## 1. 设计目标

ReadMD 的界面目标不是做成复杂的专业编辑器，而是做成一款“手机上的 Markdown 备忘录”。

核心体验优先级：

1. 能快速打开文件
2. 能清楚看见内容
3. 能快速编辑和保存
4. 能在大字模式下依然好用
5. 能在老年人也看得懂、点得到的前提下尽量减少干扰

## 2. 原型图理解

你给的原型图表达的是一套很明确的轻量应用结构：

### 首页

- 顶部左侧是应用名 `ReadMD`
- 顶部右侧是 `设置`
- 主操作只有两个：
  - `导入`
  - `新建`
- 中间有一个大搜索框
- 下方是本地笔记列表
- 每条笔记是一个可展开的卡片/条目
- 展开后显示 Markdown 内容摘要或正文片段

这说明首页的角色不是“展示一堆功能”，而是“快速找到一个文档并进入它”。

### 打开文件后的编辑/阅读页

- 顶部仍然保留 `ReadMD` 和 `设置`
- 主要操作收敛为：
  - `保存`
  - `另存`
  - `导出`
- 主体区域尽量让给正文内容
- 页面更像“单文档工作台”，而不是复杂的多栏编辑器

### 设置页

- 不是完整独立页面，更像一个轻量弹层或底部面板
- 核心只做一件事：
  - 调整字体大小
- 视觉上应尽量简洁，别做成复杂表单

## 3. 视觉语言

### 颜色

- 主色是蓝色系
- 辅助色保持很轻的浅蓝灰
- 背景以白色或近白色为主
- 强调状态时用蓝色，不要用很多杂色

### 形状

- 大量使用圆角矩形
- 边框线条细、干净
- 按钮和输入框都偏轻量，不要厚重

### 字体

- 标题明显比正文大
- 层级要清楚
- 大字模式下主要按钮和正文都要真正变大，不只是正文变大

### 留白

- 整体要有足够留白
- 首页不要太挤
- 每个区域之间要有明确间距

## 4. 页面结构准则

### 首页

必须包含：

- 应用名
- 设置入口
- 导入按钮
- 新建按钮
- 搜索框
- 笔记列表

建议规则：

- 导入和新建放第一屏显眼位置
- 搜索框足够宽
- 列表项支持展开/收起
- 首页不放太多次级功能

### 文档页

必须包含：

- 保存
- 另存
- 导出
- 正文区域

建议规则：

- 页面主体就是正文
- 工具栏尽量短
- 不要堆太多按钮

### 设置页

必须包含：

- 字体大小调节

建议规则：

- 作为轻量弹层或底部面板
- 不做复杂多级设置

## 5. 组件准则

### 按钮

- 主按钮用实心或高强调样式
- 次按钮用描边样式
- 同一页面按钮数量要少
- 按钮文字要短，最好一眼能懂

### 输入框

- 搜索框要宽
- 编辑器要大
- 边框不要太重

### 列表项

- 每个笔记项要能快速识别
- 支持展开预览
- 展开后内容只显示摘要或局部内容，避免首页太长

### 设置控件

- 字体大小优先用滑杆
- 如果还有其他阅读参数，也应该保持单项简单

## 6. 大字模式准则

大字模式不是简单把正文放大，而是整套界面一起变舒服。

必须一起变大的内容：

- 正文
- 标题
- 按钮文字
- 列表项文字
- 设置项文字

必须一起变宽的内容：

- 按钮触控区域
- 搜索框高度
- 列表项高度

不能出现的情况：

- 正文变大了，但按钮文字没变
- 按钮变大了，但页面挤成一团
- 文本变大后互相遮挡

## 7. 交互准则

- 首页第一优先级是“导入”和“新建”
- 打开文档后第一优先级是“阅读”和“保存”
- 所有重要动作都要能一眼找到
- 尽量减少二级菜单
- 少用隐藏手势
- 少用复杂侧边栏

## 8. ReadMD 应该保持的风格

关键词：

- 轻
- 清楚
- 可读
- 低学习成本
- 适老化
- 本地优先

## 9. 不建议的方向

- 复杂的工具栏
- 多层嵌套卡片
- 过度装饰
- 信息密度过高的页面
- 像代码编辑器一样的复杂布局
- 为了“专业感”堆太多按钮

## 10. 后续实现时的判断标准

以后每次做 UI 时，都可以问自己：

1. 用户能不能在 3 秒内找到导入或新建？
2. 用户能不能在一眼内找到保存？
3. 大字模式下会不会挤？
4. 首页是不是比内容本身更乱？
5. 这个控件是不是非要出现？

如果答案有一个是否定的，就该收回去。
