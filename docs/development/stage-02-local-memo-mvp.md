# ReadMD 阶段 02 开发记录：本地备忘录 MVP 闭环

日期：2026-05-24

## 阶段目标

本阶段目标是把阶段 01 中尚未完成的核心备忘录能力补齐，形成一个可以打开、阅读、编辑、保存和导出 Markdown 文件的 Android 本地应用雏形。

## 已完成

1. 系统文件选择器打开文件
   - 使用 `ACTION_OPEN_DOCUMENT` 打开 `.md`、`.markdown`、`.txt` 等文本文件。
   - 请求读取、写入和持久化 URI 权限。

2. Markdown 文件读取
   - 通过 `ContentResolver.openInputStream()` 读取文件。
   - 当前按 UTF-8 文本读取。
   - 打开失败时保留应用状态并显示提示。

3. Markdown 基础预览
   - 使用 Compose 原生组件实现轻量预览。
   - 当前支持：
     - 标题
     - 段落
     - 引用
     - 列表
     - 任务列表
     - 代码块
     - 分割线
     - 简单表格行展示
   - 加粗、斜体、删除线、行内代码当前先做文本清洗展示，后续可升级为富文本样式。

4. 编辑模式
   - 支持多行 Markdown 文本编辑。
   - 支持阅读/编辑/预览切换。
   - 未保存草稿预览不会覆盖已保存内容，可继续编辑或放弃。

5. 保存到原文件
   - 对具备写权限的 URI，使用 `ContentResolver.openOutputStream(uri, "wt")` 写回原文件。
   - 如果当前文件不可写或没有 URI，会引导另存。

6. 另存为 Markdown
   - 使用 `ACTION_CREATE_DOCUMENT` 创建新 Markdown 文件。
   - 另存成功后将新文件设为当前文件。

7. 新建备忘录
   - 支持创建内存中的新 Markdown 草稿。
   - 新草稿通过保存/另存落盘。

8. Markdown 导出
   - 使用 `ACTION_CREATE_DOCUMENT` 导出当前 Markdown 原文。
   - 导出不会改变当前编辑文件。

9. 最近打开文件
   - 使用 `SharedPreferences` 保存最近 10 个文件的 URI、展示名、打开时间和可写状态。
   - 首页/阅读页中展示最近文件列表。
   - 支持清空最近文件记录。

10. 文档内搜索
    - 支持输入关键词并统计当前文档中匹配数量。
    - 当前尚未做逐项跳转和高亮。

11. 适老化基础体验
    - 支持大字模式切换。
    - 支持字号增大/减小。
    - 主要操作按钮使用较大的 Material 组件。

12. 状态与架构
    - 增加 `ReadMDViewModel` 管理 UI 状态。
    - 增加 `DocumentRepository` 处理文件读写、最近文件和 URI 权限。
    - 增加 `DocumentState`、`RecentFile`、`ReaderSettings` 等状态模型。

## 仍未完成

1. 自动保存草稿
   - 当前草稿只保存在内存状态中。
   - 后续应落到 DataStore/SharedPreferences 或本地文件，避免进程被杀后丢失。

2. 设置页
   - 当前大字模式和字号调整只在当前运行期间生效。
   - 后续需要持久化设置。

3. 完整 Markdown 渲染
   - 当前是轻量内置渲染器。
   - 本地图片、链接点击、富文本强调、高亮、完整表格渲染仍未完成。

4. 搜索增强
   - 当前只统计匹配数量。
   - 后续可做高亮、上一个/下一个、滚动定位。

5. 测试
   - 尚未新增单元测试或 Compose UI 测试。

6. 文件编码兼容
   - 当前按 UTF-8 读取。
   - 后续可增加 BOM/GBK 等编码识别或错误提示增强。

## 本阶段暂缓内容

按照需求分析文档，以下内容仍不进入当前阶段：

- PDF 导出
- 图片长图导出
- 云同步
- 多端同步
- 插件系统
- 双链和知识图谱
- 密码保护或本地加密

## 备注

本阶段没有引入第三方 Markdown 渲染库，目的是先快速形成可用闭环并降低依赖下载成本。后续如果需要更完整的 Markdown 兼容性，可以在渲染层封装稳定后替换为成熟 Markdown 库。
