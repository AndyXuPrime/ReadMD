# ReadMD 阶段 04：真机反馈修复记录

日期：2026-05-24

## 反馈问题

1. 大字模式下按钮文字会换行挤在一起，典型场景是“关闭大字”按钮。
2. “字体”“行距”滑动条调整后，界面文字和间距没有明显变化。
3. 打开文件后还需要点“编辑”，不符合备忘录类应用的直觉。
4. 导入文件、点击“阅读”、打开 Markdown 文件时仍可能闪退，怀疑 Markdown 预览链路不稳。
5. 新打开的文件或自动恢复的草稿里，可能混入错误编码或控制字符，导致预览和编辑不稳定。
6. 真机二次反馈：打开 Obsidian 可正常显示的测试 `.md` 时，文件能打开但不会正确编译，浏览几秒后闪退。
7. 真机二次反馈：点击“新建”后会直接闪退，随后重新进入应用也可能再次闪退。
8. 界面三次反馈：夜间模式下部分区域出现黑底黑字，可读性差。
9. 界面三次反馈：预览区域偏小，不适合作为主阅读区。
10. 界面三次反馈：点击编辑时输入法占用屏幕后，编辑区剩余空间太小，不利于持续输入。
11. 界面三次反馈：缺少明确的日间/夜间切换按钮，大字模式也没有和夜间模式解耦。
12. 交互四次反馈：希望阅读页支持双指缩放字号。
13. 交互四次反馈：打开文件后底部“已打开...”提示影响阅读。
14. 交互四次反馈：缺少系统返回手势/返回键的上一界面逻辑。
15. 交互四次反馈：编辑后保存或另存，希望自动回到阅读页。

## 处理方案

1. 让按钮高度随字号和文字长度自动增长，并允许按钮文字最多两行显示。
2. 将字体滑动条绑定到首页、文档页、设置页、按钮、最近文件列表和编辑框字体。
3. 将行距滑动条绑定到页面区块间距和 Markdown 预览行距。
4. 去掉“阅读/编辑”切换按钮，文件打开后直接进入可编辑状态。
5. 预览器改为先做安全解析，再由 Compose 渲染，降低特殊 Markdown 内容导致的闪退风险。
6. 读取文件时增加编码识别、脏字符清理和草稿恢复清洗，尽量避免旧草稿把应用拖入闪退循环。
7. 预览改为按需开启，并切换到成熟的 Android Markdown 渲染库，避免首页/编辑页同时硬渲染完整预览造成设备压力。
8. 新建备忘录不再立刻写入会被强制恢复的草稿；草稿恢复只处理真正发生编辑变化的内容，减少闪退后重启循环。
9. 引入独立的 `darkMode` 设置并持久化，主题拆分为日间、夜间、大字日间、大字夜间四种配色，统一修正预览区、卡片、说明文字和 Markdown 渲染控件的前景色。
10. 文档页按原型拆分为“默认阅读页”和“编辑页”两种状态：打开文件后优先进入大预览阅读区，点击“进入编辑模式”后再进入大编辑框。
11. 编辑页去掉同屏预览，保留保存/另存/导出工具栏，把主要高度让给输入框，降低输入法弹出后的遮挡感。
12. 设置面板新增“大字模式”和“夜间模式/日间模式”两个独立切换按钮，默认仍为日间模式。
13. 在阅读页预览卡片接入双指缩放手势，将缩放倍率映射到字号设置，并限制在 0.85 到 1.8 范围内。
14. 打开文件成功后不再弹出底部 Snackbar，只在失败、保存、导出等需要反馈的动作中保留提示。
15. 接入系统返回手势/返回键：设置面板优先关闭，编辑页返回阅读页，阅读页返回首页。
16. 保存或另存成功后自动切回阅读页，便于立即检查 Markdown 渲染效果。

## 待真机复测

- Android 10/12 上导入本地 `.md` 文件是否仍闪退。
- 大字模式下“关闭大字”“保存/另存/导出”等按钮是否仍有文字重叠。
- 字体与行距滑动条是否能即时改变界面观感。
- 新建文件后重启应用，是否还会被草稿恢复带回闪退循环。
- 夜间模式下首页、阅读页、编辑页、设置面板和 Markdown 预览是否都保持“深底浅字”，不存在黑底黑字。
- 阅读页预览区是否已明显放大，长文档浏览是否比之前顺手。
- 编辑页在弹出输入法后，输入框是否仍能看到正在输入的文字和上下文。
- 阅读页双指缩放字号是否灵敏且不会影响滚动阅读。
- 打开文件后底部是否不再出现“已打开...”提示。
- 系统返回手势是否符合“设置关闭、编辑回阅读、阅读回首页”的顺序。
- 保存或另存成功后是否自动回到阅读页。
