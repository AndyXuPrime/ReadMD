# ReadMD 阶段 04：真机反馈修复记录

日期：2026-05-24
最近更新：2026-05-25

## 1. 阶段背景

阶段 04 的核心任务不是继续堆功能，而是围绕用户在真机上的连续反馈，集中修复“能不能稳定打开、能不能顺手阅读、能不能安心编辑保存”这三件事。

这一阶段的特点是：

- 问题并非集中在单一模块，而是同时出现在文件读取、Markdown 预览、草稿恢复、主题配色、阅读布局和返回逻辑上。
- 很多问题只有在真机上才能稳定复现，尤其是 Android 10/12、输入法弹出、大字模式、夜间模式和长 Markdown 文档阅读场景。
- 交互方案在阶段中途发生过一次明显调整：早期曾尝试“打开后直接进入可编辑状态”，后来根据阅读优先原型和真机反馈，改为“打开后默认进入阅读页，点击进入编辑模式；保存或另存后自动回阅读页”。

## 2. 反馈来源与问题演进

### 第一轮反馈：基础可用性问题

首轮真机反馈主要集中在“能不能稳定使用”：

1. 大字模式下按钮文字会挤在一起，典型场景是“关闭大字”按钮。
2. “字体”“行距”滑动条调节后，界面文字和间距没有明显变化。
3. 打开文件后还需要再点“编辑”，不够像备忘录。
4. 导入文件、点击阅读、打开 Markdown 文件时仍可能闪退。

### 第二轮反馈：闪退与预览链路问题

在 Android 10/12 真机上，问题进一步收敛到文档读取和预览链路：

1. Obsidian 可正常打开的测试 Markdown，在 ReadMD 中无法正确渲染，浏览几秒后闪退。
2. 点击“新建”后会直接闪退，重新进入应用后还可能再次闪退。
3. 自动恢复的旧草稿可能把应用带回闪退循环。

### 第三轮反馈：阅读体验与主题问题

当闪退问题初步缓解后，新的问题集中暴露在阅读体验上：

1. 夜间模式下部分区域出现黑底黑字，可读性差。
2. 预览区域偏小，不适合作为主阅读区。
3. 点击编辑后输入法占用大量屏幕空间，编辑区剩余空间太小。
4. 缺少明确的日间/夜间切换入口，大字模式也没有和夜间模式解耦。

### 第四轮反馈：交互细节问题

在基础链路基本打通后，反馈开始集中在细节交互：

1. 希望阅读页支持双指缩放字号。
2. 打开文件后底部“已打开...”提示影响阅读。
3. 缺少系统返回手势/返回键的上一界面逻辑。
4. 编辑后保存或另存，希望自动回到阅读页。

### 第五轮反馈：阅读页性能与手势问题

2026-05-25 的最新反馈指出：

1. 阅读页单指滑动浏览长 Markdown 文档时卡顿明显。
2. 双指缩放功能虽然代码中已接入，但真机上几乎感觉不到效果。

这一轮反馈说明问题已经从“有没有功能”转向“功能是否真正可用”。

### 第六轮反馈：缩放影响范围与滚动回归

2026-05-26 的反馈进一步指出：

1. 阅读页双指缩放会影响首页字号，导致返回首页后列表和按钮也被放大。
2. 放大到约 160% 时，部分较大的 Markdown 标题或文字会出现重叠，阅读区显得混乱。
3. 最新版阅读页可以缩放，但又出现无法上下滑动阅读全文的问题。

这说明阅读缩放需要从“全局字号设置”中拆出来，变成阅读页自己的临时阅读参数；同时手势识别不能拦截单指滚动。

## 3. 问题定位与分析

### 3.1 文件打开与闪退

阶段 04 前半段的闪退，主要来自三类风险叠加：

- 文件内容里存在 BOM、空字符、异常换行或历史脏字符。
- 草稿恢复逻辑过早介入，把异常内容反复恢复进当前会话。
- Markdown 预览链路不够稳，复杂内容在真机上容易触发 UI 压力或异常。

处理方向不是单点修补，而是把“文件读取 -> 内容清洗 -> 草稿恢复 -> 预览渲染”整条链路逐步收紧。

### 3.2 夜间模式与大字模式

问题根源不只是颜色没改，而是主题状态没有被完整传导到所有阅读相关组件，包括：

- 首页卡片和说明文字
- 阅读页预览容器
- 编辑页输入区
- Markdown 预览控件内部前景色和链接色

同时，大字模式此前更多只作用于部分正文，没有覆盖按钮高度、触控面积和设置项字号。

### 3.3 阅读页卡顿与双指缩放不明显

这一轮定位出来的核心原因有两个：

1. 阅读页此前把 `detectTransformGestures` 挂在外层 Compose 卡片上，内部真正显示 Markdown 的却是 `AndroidView` 承载的原生 `TextView`。两层触摸处理容易互相争抢，导致单指滚动发黏，而双指缩放也很难稳定命中。
2. 预览控件在 `update` 时会重复执行 `markwon.setMarkdown(...)`。这意味着哪怕只是字号变化、主题变化或轻微重组，也可能触发整段 Markdown 再次解析和重绘，长文档场景下很容易出现阅读卡顿。

这类问题不是“再调大缩放倍率”能解决的，必须把手势和渲染更新路径一起改掉。

### 3.4 阅读缩放影响首页

阅读页手势缩放最初复用了全局 `fontScale`。这样实现简单，但副作用很明显：用户为了读清当前文档临时放大字号，首页、最近文件列表、设置面板等非阅读区域也会被同步放大，容易造成布局拥挤。

产品定位上，首页承担的是“找文件”和“进入文档”，不应该被某篇文档的临时阅读习惯影响。因此阅读页手势缩放应当只影响当前阅读预览。

### 3.5 放大上限与文字重叠

真机反馈显示，约 160% 时较大标题容易出现重叠。结合移动端阅读场景，阶段 04 将最大阅读缩放收敛到 155%：

- 155% 比 150% 多一点阅读余量。
- 155% 避开了用户已经观察到的 160% 混乱区间。
- 大字模式仍可通过独立开关提供整体适老化体验，不需要把普通手势缩放推得过高。

## 4. 主要修复动作

### 4.1 文件与草稿链路

1. 文件读取增加编码识别、BOM 清理、空字符与异常换行清洗。
2. 草稿恢复只针对真实发生过编辑变更的内容，减少旧草稿把应用拖回闪退循环。
3. 新建备忘录不再过早写入会被强制恢复的初始草稿。

### 4.2 Markdown 预览链路

1. 预览能力切换到成熟的 Android Markdown 渲染方案。
2. 阅读页和编辑页解耦，避免同屏同时承载大编辑框和完整预览。
3. 预览内容增加安全清洗和超长内容截断说明，降低超长文档的瞬时压力。

### 4.3 阅读与编辑页面结构

1. 文档页拆分为“阅读页”和“编辑页”两种状态。
2. 打开文件后优先进入阅读页，把主要空间留给 Markdown 预览。
3. 编辑页去掉同屏预览，保留保存、另存、导出工具栏，把主要高度交给输入框。
4. 保存或另存成功后自动返回阅读页，便于立即检查渲染效果。

### 4.4 主题、字号与适老化

1. 引入独立的 `darkMode` 设置并持久化。
2. 将主题拆分为日间、夜间、大字日间、大字夜间四种组合。
3. 字体和行距滑动条绑定到首页、阅读页、编辑页、设置页、按钮和最近文件列表。
4. 按钮高度随字号和文字长度自动增长，避免大字模式下文字叠在一起。

### 4.5 交互细节

1. 打开文件成功后不再弹出“已打开...”底部提示，只在失败、保存、导出等需要反馈的动作中保留提示。
2. 接入系统返回手势/返回键逻辑：设置面板优先关闭，编辑页返回阅读页，阅读页返回首页。
3. 设置面板新增“大字模式”和“夜间模式/日间模式”两个独立切换按钮。

### 4.6 本轮新增：阅读页滚动与双指缩放修复

2026-05-25 针对阅读页卡顿和双指缩放不明显，追加了以下处理：

1. 移除阅读页外层 Compose 卡片上的缩放手势监听，避免与内部原生预览控件争抢单指滚动事件。
2. 将双指缩放手势下沉到 Markdown 预览内部的原生 `TextView`，使用原生缩放检测器处理双指手势。
3. 预览控件改为“内容变化时才重新执行 Markdown 渲染”，仅字号、行距、主题变化时不再重复整段解析。
4. 阅读页增加当前阅读字号百分比提示，让双指缩放是否生效可以被直接观察到。

这一轮修复的目标不是增加新功能，而是把“能滚得动、缩得出来、缩放后看得见变化”这三个体验问题一次收紧。

### 4.7 本轮补充：阅读缩放隔离与滚动修复

2026-05-26 追加处理：

1. 新增阅读页独立缩放状态，双指缩放只改变当前阅读预览，不再写入全局字体设置。
2. 全局字体滑动条和阅读手势缩放的最大值统一收敛到 155%，减少大标题和大字模式组合时的重叠风险。
3. Markdown 预览在大字号时强制保留更安全的最小行距，降低文字上下挤压。
4. 修复原生缩放检测器吞掉单指触摸事件的问题，让单指上下滑动重新交给 `TextView` 处理。

## 5. 当前结果

截至 2026-05-25，阶段 04 已经把以下关键路径基本打通：

- 本地 Markdown 文件可导入、阅读、编辑、保存、另存和导出 Markdown 原文。
- 阅读页与编辑页分离，主流程更符合“先阅读、后编辑”的定位。
- 夜间模式与大字模式可独立切换，并已覆盖主要阅读界面。
- 新建、草稿恢复、返回逻辑和成功提示路径比阶段早期稳定许多。
- 阅读页双指缩放已从“代码存在但不明显”收紧为更贴近原生行为的实现。
- 阅读缩放已和首页字号解耦，返回首页后首页布局不再跟随文档阅读缩放变化。

## 7. 待真机复测

以下项目仍建议继续由真机手动验证，重点观察 Android 10/12 和长文档场景：

- 导入本地 `.md` 文件后是否仍出现闪退。
- 新建文件后重启应用，是否还会被草稿恢复带回异常状态。
- 夜间模式下首页、阅读页、编辑页、设置面板和 Markdown 预览是否都保持“深底浅字”。
- 大字模式下“关闭大字”“保存”“另存”“导出”等按钮是否仍有文字重叠。
- 阅读页长文档单指滚动是否明显比之前顺滑。
- 阅读页双指缩放字号是否灵敏、稳定，并且缩放结果能从界面上直接看出来。
- 阅读页缩放后返回首页，首页按钮、列表和搜索框是否保持原本大小。
- 放大到 155% 附近时，较大的 Markdown 标题是否仍保持可读、不重叠。
- 编辑页弹出输入法后，输入框是否仍能看到正在输入的文字和上下文。
- 系统返回手势是否符合“设置关闭、编辑回阅读、阅读回首页”的顺序。
- 保存或另存成功后是否自动回到阅读页。

## 8. 阶段结论

阶段 04 的价值不在于又加了多少新功能，而在于把 ReadMD 从“可以演示”往“真机可持续使用”推进了一大步。

这一阶段说明了两件事：

1. 对移动端 Markdown 应用来说，稳定的文件读取、稳妥的草稿恢复、顺滑的阅读滚动和清楚的主题适配，比堆更多高级功能更重要。
2. 真机反馈非常关键。很多问题在本地构建和代码阅读中看不出来，只有在具体设备、具体系统版本、具体输入法和真实 Markdown 文件上，才会暴露出真正影响体验的细节。
