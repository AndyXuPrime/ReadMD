## 1. 项目背景

ReadMD 是一款面向 Android 手机的轻量级 Markdown 备忘录与文件浏览应用。它的核心价值是：让用户可以在手机上方便地打开、阅读、编辑、保存和导出 `.md` 文件。

当前大量 LLM、知识库、笔记工具、开发文档和课程资料都使用 Markdown 格式输出，但普通手机用户往往缺少一个简单、稳定、适合移动端阅读的 Markdown 浏览和编辑工具。ReadMD 应定位为“手机上的 Markdown 备忘录”，兼顾文件浏览、Markdown 阅读、快速编辑、导出分享和适老化大字阅读。

## 2. 产品定位

### 2.1 一句话定位

ReadMD 是一款简单易用、支持大字阅读的 Android Markdown 文件浏览与编辑备忘录应用。

### 2.2 核心目标

1. 让用户能通过 Android 系统文件选择器选择并打开 Markdown 文件。
2. 让 Markdown 内容在手机上正确渲染，具备舒适的阅读体验。
3. 让用户能直接编辑 Markdown 文件并保存。
4. 让用户能将笔记导出为 Markdown 文件；PDF 和图片导出作为后续增强，不进入首版。
5. 让老年用户、低视力用户也能清晰阅读和操作。

### 2.3 非目标

ReadMD 当前定位为本地 Android 备忘录应用，不追求完整替代 Obsidian。当前及可预见版本不考虑双链、知识图谱、插件系统、云同步、多端同步、多人协作、Git 同步、复杂主题市场、密码保护或本地加密笔记。产品应保持简单、本地、轻量，优先保证基础备忘录能力稳定可用。

### 2.4 当前实现状态（截至 2026-05-25）

本文是需求分析与规划文档，部分内容用于描述目标能力和后续增强方向，不完全等同于当前 APK 已实现能力。当前项目的实际状态如下：

已实现：

- Android 端本地应用，最低目标支持 Android 8.0，并按 Android 10 及以上分区存储机制使用系统文件选择器。
- 支持通过系统文件选择器导入 `.md`、`.markdown`、`.txt` 文件。
- 支持新建本地 Markdown 备忘录。
- 打开文档后默认进入阅读页，阅读页使用 Markwon 渲染常见 Markdown 内容。
- 支持进入编辑页修改原始 Markdown 文本。
- 支持保存到原文件、另存为新文件、导出 Markdown 原文。
- 支持最近打开文件、草稿恢复、字号调整、行距调整、大字模式、日间/夜间模式。
- 阅读页支持双指缩放字号；系统返回手势支持设置关闭、编辑回阅读、阅读回首页。

未实现或暂不作为当前目标：

- PDF、JPG、PNG 导出尚未实现，当前只导出 Markdown 原文。
- 当前不做 iOS 端、云同步、多端同步、插件系统、双链、知识图谱、密码保护或本地加密。
- 文内搜索、长文滚动位置记忆、屏幕常亮开关、本地图片完整预览等仍属于后续可选增强。
- 当前默认不主动加载网络资源；本地图片支持以实际渲染能力为准，不作为首版稳定验收项。

## 3. 用户画像

### 3.1 普通手机用户

需求：

- 打开聊天软件、网盘、浏览器下载到手机中的 `.md` 文件。
- 像备忘录一样查看和修改文本内容。
- 将内容导出为 Markdown 文件，或在后续版本导出为 PDF/图片后分享给他人。

痛点：

- 手机自带文件管理器通常不能正确预览 Markdown。
- 普通文本编辑器能打开，但阅读体验差。
- Markdown 语法对非技术用户不够直观。

### 3.2 LLM 高频用户

需求：

- 保存 AI 生成的 Markdown 文档、总结、计划、代码说明。
- 在手机上阅读长文本结构化内容。
- 对 AI 生成内容进行二次编辑和整理。

痛点：

- AI 输出常带标题、列表、代码块、表格，普通文本查看很乱。
- 手机端缺少“复制为 Markdown 后直接管理”的轻量工具。

### 3.3 学生、教师、知识工作者

需求：

- 阅读课程笔记、复习资料、会议记录、项目文档。
- 快速搜索和整理多个 Markdown 文件。
- 将文档导出为 Markdown 文件，后续可支持 PDF 或图片分享。

痛点：

- 移动端文档阅读工具大多偏 Office/PDF，不够适配 Markdown。
- 需要在碎片时间阅读和补充内容。

### 3.4 老年用户或低视力用户

需求：

- 字体大、对比度高、按钮清晰。
- 操作步骤少，避免复杂层级。
- 支持一键切换“大字模式”。

痛点：

- 普通编辑器字体小、按钮密、误触多。
- 复杂文件路径和权限弹窗难理解。

## 4. 使用场景

### 4.1 打开本地 Markdown 文件

用户在 ReadMD 中点击“打开文件”，通过 Android 系统文件选择器选择 `.md` 文件，应用读取内容并进入阅读模式。

验收标准：

- 能通过 Android 系统文件选择器选择 `.md`、`.markdown`、`.txt` 文件。
- 成功打开后显示文件名、正文内容和最近打开记录。
- 文件不存在、权限失效或编码异常时给出可理解提示。

### 4.2 阅读 Markdown

用户打开文件后，应用将 Markdown 渲染为标题、段落、列表、引用、代码块、表格、链接、图片等结构。

验收标准：

- 支持常见 Markdown 语法渲染。
- 长文滚动流畅。
- 标题层级清晰，代码块可横向滚动或自动换行。
- 支持亮色、暗色和大字模式。

### 4.3 编辑 Markdown

用户点击“进入编辑模式”等编辑入口后进入编辑页，修改内容后保存到原文件，或另存为新文件。

验收标准：

- 支持纯文本编辑 Markdown。
- 支持保存、另存为等基础操作。
- 编辑前后能在“编辑页”和“阅读页”之间切换。
- 保存失败时不丢失用户输入。

### 4.4 新建备忘录

用户可以在应用内新建一个 Markdown 备忘录，输入标题和正文，并保存到用户选择的目录。

验收标准：

- 支持新建空白 Markdown 文件。
- 默认文件名可根据标题或时间生成。
- 用户能选择保存位置。

### 4.5 导出和分享

用户可以将当前 Markdown 文档导出为 `.md` 文件。PDF 和图片导出属于后续本地导出增强，不要求首版完成。

验收标准：

- 首版支持导出 Markdown 原文文件。
- 后续可支持导出 PDF，保留标题、列表、代码块等基本格式。
- 后续可支持导出长图或分页图片，用于社交软件分享。
- 导出完成后可调用 Android 分享面板。

### 4.6 大字阅读模式

用户或家人可在设置中开启“大字模式”，应用整体使用更大的字号、更高对比度和更简单的操作界面。

验收标准：

- 大字模式下正文、标题、按钮、菜单字号明显增大。
- 主要按钮触控区域不小于 48dp，建议 56dp 以上。
- 操作路径保持简单，不依赖复杂手势。
- 支持跟随系统字体缩放。

## 5. 功能需求

### 5.1 文件访问与管理

优先级：P0

功能点：

- 使用 Android Storage Access Framework 选择单个 Markdown 文件。
- 首版采用系统文件选择器，不内置复杂文件管理器。
- 支持最近打开文件列表。
- 支持显示文件名、路径来源、最后打开时间。
- 支持文件读取异常提示。

建议：

- Android 10 及以上应遵循分区存储机制，避免直接申请过宽的存储权限。
- 对外部文件使用 `content://` URI 持久化授权。
- 最近文件只保存 URI、展示名、时间等元数据，不复制用户文件，除非用户主动导入。

### 5.2 Markdown 渲染

优先级：P0

需要支持：

- 标题：`#` 到 `######`
- 段落和换行
- 加粗、斜体、删除线
- 有序列表、无序列表、任务列表
- 引用块
- 行内代码和代码块
- 链接
- 本地图片
- 表格
- 分割线

增强支持：

- 目录导航
- 代码高亮
- 脚注，可暂不做

MVP 阶段建议先支持基础 Markdown、表格、任务列表和代码块。本地图片可作为后续兼容性增强；默认不加载网络图片，降低隐私和网络依赖风险。

### 5.3 阅读体验

优先级：P0

功能点：

- 阅读模式作为默认打开状态。
- 支持字体大小调节。
- 支持行距调节。
- 支持亮色、暗色、大字高对比主题。
- 支持屏幕常亮开关。
- 支持长文滚动位置记忆。
- 支持文内搜索。

适配要求：

- 手机竖屏优先。
- 横屏可用，但不是 MVP 的重点。
- 小屏手机不能出现正文被按钮遮挡的问题。

### 5.4 编辑能力

优先级：P0

功能点：

- 编辑原始 Markdown 文本。
- 支持保存到原文件。
- 支持另存为新文件。
- 支持新建 Markdown 文件。
- 支持编辑与预览切换。
- 支持未保存变更提醒。
- 支持自动保存草稿，避免异常退出丢内容。

建议编辑模式：

- MVP：纯文本编辑 + 单独预览。
- V1.1：分屏编辑/预览，适配大屏或横屏。
- 所见即所得编辑暂不纳入当前规划，先保证备忘录式文本编辑稳定可靠。

### 5.5 导出能力

优先级：P1，其中 Markdown 原文导出为首版目标；PDF 和图片导出为后续增强。

导出 Markdown：

- 可导出当前文档为 `.md`。
- 可选择保存目录。
- 文件名冲突时提示覆盖、重命名或取消。

导出 PDF：

- 不进入首版。
- 将渲染后的 Markdown 内容转换为 PDF。
- 支持 A4 页面或移动阅读版页面。
- 支持标题、列表、表格、代码块基本样式。
- 支持导出完成后预览或分享。

导出图片：

- 不进入首版。
- 支持导出当前全文长图。
- 长文过长时应支持分页图片，避免单张图片过大失败。
- 支持导出为 `.jpg` 或 `.png`，建议优先 `.png` 保证文字清晰。

### 5.6 适老化与无障碍

优先级：P0

设计原则：

- 看得清：大字号、高对比、清晰层级。
- 点得到：按钮大、间距足、避免密集操作。
- 听得懂：提示语使用生活化中文。
- 退得回：关键操作有确认，保存失败有恢复路径。

具体要求：

- 支持“大字模式”入口，建议放在首页明显位置和设置页。
- 正文基础字号建议：普通模式 16sp 到 18sp，大字模式 22sp 到 28sp。
- 按钮触控区域最小 48dp，大字模式建议 56dp 到 64dp。
- 所有图标按钮需要有文字标签或无障碍描述。
- 支持 TalkBack 的内容描述。
- 颜色对比度应满足 WCAG AA 级别。
- 避免只用颜色传达状态，例如错误状态同时显示文字说明。

### 5.7 搜索与组织

优先级：P2

功能点：

- 当前文档内搜索。
- 最近文件搜索。
- 已授权文件夹内按文件名搜索。
- 可选：收藏文件。
- 可选：标签或文件夹分类。

搜索能力属于后续增强；当前首页已支持最近文件搜索，文内搜索暂不作为当前版本验收项。

### 5.8 设置

优先级：P1

功能点：

- 字体大小
- 行距
- 主题：跟随系统、亮色、暗色、大字高对比
- 默认打开模式：阅读或编辑
- 自动保存开关
- 导出默认格式
- 清除最近文件记录

## 6. 非功能需求

### 6.1 性能

- 打开普通文档应在 1 秒内进入可读状态。
- 1MB 到 5MB 的 Markdown 长文应可正常滚动和编辑。
- 渲染大文件时避免阻塞主线程。
- 图片加载应有占位和失败提示。

### 6.2 稳定性

- 编辑内容需要草稿保护。
- 文件权限失效时提示重新授权。
- 导出失败时保留原内容，并说明失败原因。
- 应避免因单个非法 Markdown 或图片链接导致应用崩溃。

### 6.3 隐私与安全

- 默认不上传用户文档。
- 不要求账号登录。
- 不收集正文内容。
- 对外分享前由用户主动确认。
- 默认只加载本地图片，不加载网络图片，避免隐私泄露和网络依赖。

### 6.4 兼容性

- 最低支持 Android 8.0。
- 必须确保 Android 9、10、11、12、13、14 及更高版本正常使用。
- 重点适配 Android 10 及以上的分区存储，避免依赖旧式全盘存储权限。
- 支持主流中文 Android 系统，如 MIUI、HarmonyOS 兼容 Android、ColorOS、OriginOS 等。

### 6.5 可维护性

- 文件访问、Markdown 渲染、编辑状态、导出逻辑应模块化。
- 渲染库与导出实现应封装，方便后续替换。
- 适老化主题应通过统一设计令牌控制字号、颜色、间距。

## 7. 信息架构

建议首版页面：

1. 首页
   - 打开文件
   - 新建备忘录
   - 最近打开
   - 大字模式开关

2. 阅读页
   - 顶部：返回、文件名、更多操作
   - 正文：Markdown 渲染内容
   - 底部或悬浮操作：编辑、搜索、导出

3. 编辑页
   - 顶部：返回、保存、预览
   - 正文：Markdown 文本编辑器
   - 底部：格式辅助工具栏，可后续实现

4. 导出页或导出弹窗
   - Markdown
   - PDF，后续增强
   - 图片，后续增强
   - 分享

5. 设置页
   - 字体与主题
   - 编辑与自动保存
   - 最近文件管理
   - 隐私设置

## 8. 关键流程

### 8.1 打开文件流程

1. 用户点击“打开文件”。
2. 系统文件选择器弹出。
3. 用户选择 `.md` 文件。
4. 应用获取 URI 授权。
5. 读取文件内容并检测编码。
6. 渲染 Markdown。
7. 写入最近打开记录。

异常处理：

- 未选择文件：返回首页。
- 读取失败：提示“无法打开该文件，请检查文件是否存在或重新选择”。
- 文件过大：提示可能加载较慢，并允许继续。

### 8.2 编辑保存流程

1. 用户进入编辑模式。
2. 修改内容。
3. 应用定时保存草稿。
4. 用户点击保存。
5. 应用写回原 URI 或提示另存为。
6. 保存成功后更新预览。

异常处理：

- 原文件不可写：提示另存为。
- 保存失败：保留编辑内容，允许重试或导出。
- 返回时存在未保存内容：提示保存、放弃或继续编辑。

### 8.3 导出 Markdown 流程

1. 用户点击导出。
2. 选择 Markdown。
3. 应用准备当前文档原始 Markdown 内容。
4. 用户选择保存位置和文件名。
5. 导出成功后显示分享入口。

说明：

- PDF 导出实现成本高于 Markdown 原文导出，涉及分页、样式、表格、代码块和图片排版，首版不要求完成。
- PDF 和图片导出可在核心读写体验稳定后再作为本地导出增强实现。

### 8.4 大字模式流程

1. 用户在首页或设置页开启大字模式。
2. 应用立即切换字号、间距、按钮尺寸和高对比主题。
3. 该设置持久保存。
4. 下次启动默认沿用。

## 9. MVP 范围建议

### 9.1 MVP 必做

说明：本节是初始 MVP 范围建议，当前 APK 的实际完成情况以 2.4 节和阶段开发记录为准。

- 首页：打开文件、新建文件、最近打开。
- 文件读取：通过系统文件选择器打开 `.md`。
- Markdown 渲染：基础语法、表格、任务列表、代码块；本地图片作为后续兼容性增强，以实际渲染能力为准。
- 编辑：纯文本编辑、保存、另存为、未保存提醒。
- 阅读体验：字体大小、亮色/暗色、大字模式。
- 导出：Markdown 原文导出。
- 基础无障碍：大触控区域、TalkBack 描述、跟随系统字体。

### 9.2 MVP 可延后

- PDF 导出。
- 图片长图导出。

### 9.3 明确不做

- iOS 端。
- 双链和知识图谱。
- 插件系统。
- 云同步。
- 多端同步。
- 云盘目录专项适配。
- 密码保护或本地加密笔记。
- 网络图片默认加载。

### 9.4 推荐版本路线

v0.1 原型：

- 创建 Android 项目。
- 实现文件选择、读取、基础渲染。
- 首页和阅读页可用。

v0.2 MVP：

- 新建和编辑 Markdown。
- 保存、另存为、最近文件。
- 大字模式。

v0.3 导出增强：

- Markdown 导出。
- 分享面板。

v0.4 图片和阅读增强：

- 长图导出。
- 文内搜索。
- 目录导航。

v0.5 PDF 导出增强：

- PDF 导出。
- 导出样式优化。
- 导出失败兜底提示。

v1.0 稳定版：

- 完整设置页。
- 更好的异常处理。
- 性能优化。
- 适老化体验验收。

## 10. 技术建议

### 10.1 Android 技术栈

建议：

- 语言：Kotlin
- UI：Jetpack Compose
- 架构：MVVM
- 状态管理：ViewModel + StateFlow
- 本地存储：DataStore 保存设置，Room 保存最近文件元数据
- 文件访问：Storage Access Framework
- Markdown 渲染：Markwon 或 CommonMark 相关库
- 导出 Markdown：Storage Access Framework 创建文档并写入原始 Markdown 内容
- 后续 PDF 导出：Android Print/PdfDocument 或 WebView 渲染后打印

说明：

- 如果目标是快速做出美观的 Android 原生应用，Jetpack Compose 更适合。
- 如果团队更熟悉传统 View，也可以用 XML + Fragment，但后续适老化主题和动态字号维护成本会更高。
- 最低支持 Android 8.0，同时需要持续验证更高 Android 版本下的文件选择、保存和分享流程。

### 10.2 模块划分建议

可按如下方向组织：

- `app`：应用入口和页面。
- `core-file`：文件选择、读取、写入、授权管理。
- `core-markdown`：Markdown 解析和渲染适配。
- `core-export`：Markdown 导出，后续扩展 PDF、图片导出。
- `core-design`：主题、字号、间距、组件。
- `data`：最近文件、设置、草稿数据。

如果项目初期规模较小，也可以先保持单模块，但包结构应按上述边界划分。

## 11. 验收标准总表

P0 验收：

- 用户可以从系统文件选择器打开 Markdown 文件。
- Markdown 基础语法能正确渲染。
- 用户可以编辑并保存 Markdown。
- 用户可以新建 Markdown 备忘录。
- 应用支持大字模式。
- 应用不会在常见读取、保存失败场景下丢失用户编辑内容。

P1 验收：

- 用户可以导出 Markdown。
- 用户可以调整字号、主题和行距。
- 最近文件可用。
- 文档内搜索可用。

P2 验收：

- 用户可以导出长图。
- 用户可以导出 PDF。
- 支持目录导航。
- 支持收藏或标签。

## 12. 风险与对策

### 12.1 Android 文件权限复杂

风险：

- 不同 Android 版本和国产系统文件授权行为不同。

对策：

- 优先使用 Storage Access Framework。
- 保存 URI 持久化权限。
- 对不可写文件提供另存为路径。

### 12.2 Markdown 渲染范围过大

风险：

- Markdown 方言很多，公式、图表、HTML、脚注等支持成本高，不适合放入备忘录首版。

对策：

- MVP 明确支持 CommonMark + 表格 + 任务列表。
- 高级语法默认不做，除非后续用户需求明确。

### 12.3 导出 PDF 和长图质量不稳定

风险：

- 长文分页、表格、代码块和图片导出容易出现样式问题。

对策：

- 先实现 Markdown 原文导出。
- PDF 导出优先保证可读性。
- 长图导出设置最大高度，超出自动分页。

### 12.4 大字模式不是简单放大

风险：

- 字变大后按钮、导航、弹窗可能挤压或遮挡内容。

对策：

- 从设计系统层面定义大字模式。
- 所有页面做大字模式验收。
- 避免密集工具栏和复杂菜单。

## 13. 已确认产品边界

1. 当前只做 Android 端，不考虑 iOS。
2. 最低支持 Android 8.0，并确保更高 Android 系统版本正常使用。
3. 文件访问采用 Android 系统文件选择器，不内置复杂文件管理器。
4. PDF 导出实现相对更复杂，首版不要求完成。
5. 不支持云同步，也不把云同步作为未来规划。
6. 不做多端同步。
7. 默认只加载本地图片，不加载网络图片。
8. 不做密码保护或本地加密笔记，也不作为未来规划。
9. 不做双链、知识图谱和插件系统。

## 14. 结论

ReadMD 的首要价值不是做一个复杂知识库，而是补齐手机端 Markdown 备忘录的基础体验。建议第一阶段聚焦“打开文件、正确渲染、本地编辑、大字阅读、保存不丢失、Markdown 导出”六件事，把它做稳定、做清楚。PDF 和长图导出可以在核心读写体验稳定后逐步增强，但不应影响首版交付。
